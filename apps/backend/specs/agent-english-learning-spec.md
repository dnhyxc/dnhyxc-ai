# Agent 英语学习增强能力 SPEC（面向大众人群）

> **关联模块**：`apps/backend/src/services/agent`（LangChain Agent、工具调用、SSE、会话记忆等）。  
> **文档性质**：产品与后端行为规格说明，用于评审与迭代排期；**本文档撰写时不修改业务源码**。  
> **版本**：v1 — 覆盖单词 / 短语 / 句子 / 语法、名著与文献导读、按需学习资料、实时翻译，并给出「大部分人可用」的分层与边界。

---

## 1. 目标与范围

### 1.1 目标

- 在现有 **通用 Agent + RAG（知识库检索）+ 互联网检索** 能力之上，定义一套 **英语学习（English learning）** 专项行为：让用户能以自然语言完成 **词汇、短语、句子、语法** 的学习与练习；能接触 **著名英语文献（节选 / 导读 / 公版优先）**；能根据用户当下需求 **动态获取可读的学习材料**（模型生成 + 可选联网补充）；并提供 **实时翻译（中英互译、对照呈现）**，适配 **大部分人群** 的典型水平差异。

### 1.2 范围（本 SPEC 覆盖）

| 维度 | 说明 |
|------|------|
| 学习者画像 | 中小学生家长陪学、大学生、职场通用需求；不以某一考试为唯一叙事，但可兼容「四级 / 雅思口语」等用户自述目标 |
| 内容形态 | 单词卡片式讲解、短语搭配、句型与改写、语法要点与常见误区、分级短文、口语对话范例 |
| 文献与名著 | 以 **导读、节选、摘要、主题讨论** 为主；优先 **公版（public domain）** 或普遍见于教材的节选；完整长篇搬运需遵守版权与引用规范 |
| 动态资料 | 用户指定主题 / 难度 / 场景时，由大模型生成练习材料；必要时通过 **联网检索** 获取公开网页、新闻、百科类摘要作为补充阅读 |
| 翻译 | 对话内 **实时** 流式输出译文；支持整段翻译、逐句对照、术语简要注释 |

### 1.3 非目标（避免过度承诺）

- **不替代**：专业同声传译、法律 / 医学等需签字负责的认证译文；外教一对一纠音；学校正式阅卷与评分。
- **不默认**：自动抓取并存储用户未授权的受版权全文作品。
- **不在本 SPEC 强制**：单独新建「英语学习」微服务或独立路由（可与现有 Agent 入口共用，由专项模式或系统提示区分，属实现阶段决策）。

## 2. 术语（中英对照）

- **CEFR（欧洲语言共同参考框架）**：A1～C2 等级，用于描述难度；产品可对用户用「初中 / 高中 / 四级 / 能日常交流」等映射。
- **RAG（检索增强生成，Retrieval-Augmented Generation）**：从用户已入库的笔记、生词本、摘录中检索后再生成回答。
- **SSE（Server-Sent Events）**：服务端流式推送模型输出，用于「实时翻译与讲解」的打字机体验。
- **Tool calling（工具调用）**：Agent 调用「知识库检索」「互联网搜索」等工具；本场景下用于补充时效资料与核实用法。

## 3. 大众人群能力分层（产品预设）

设计功能与默认输出时，应允许用户 **自述水平** 或 **选择档位**，避免「过难导致放弃」或「过简导致无聊」。建议默认三档（实现时可映射到 prompt 与篇幅参数）。

| 档位 | 典型用户 | 内容特征 | 输出节奏 |
|------|----------|----------|----------|
| **基础** | 词汇量有限、需要中考 / 日常 survival English | 短句、高频词、中文释义比例略高、例句极短 | 每次 1 个主点 + 2 条例句 |
| **进阶** | 高中～四级、工作邮件刚需 | 搭配与辨析、复合句、小型段落精读 | 每次 2～3 个知识点 + 小练习 |
| **提高** | 六级 / 考研 / 雅思起步 | 语篇、修辞、近义词细微差别、文献导读 | 可输出摘要 + 讨论题 |

**规则**：若用户未说明水平，Agent 应用 **一两句澄清问题**（当前用途、大致年级或考试目标），再给出内容；避免假设所有人同一难度。

## 4. 功能需求详述

### 4.1 单词学习（Vocabulary）

- **输入**：目标词、主题（如「商务邮件」「旅游」）、是否要与中文母语易混词对比。
- **输出（建议结构）**：音标（IPA，国际音标）/ 词性 / 核心义项 / 常见搭配 / 1～3 个例句 / 可选「易混词对比」。
- **复习**：可生成「英译中 / 中译英」微型问答或填空线索（不要求在本 SPEC 内单独持久化错题本表，但产品可规划）。

### 4.2 短语与固定表达（Phrases & chunks）

- **覆盖**：动词短语、介词搭配、习语（idiom）在大众读物中的常见度分级说明。
- **输出**：含义、使用语境、正式 / 口语色彩标注、替换说法（如有）。

### 4.3 句子层面（Sentences）

- **功能**：句型拆解、改写（正式 ↔ 口语）、合并与拆分、连接词逻辑。
- **练习**：给出一句话，请用户自译或改错；Agent 给予分步反馈（先结构后词汇）。

### 4.4 语法（Grammar）

- **范围**：时态、主谓一致、从句、非谓语、虚拟语气等 **高频考点与真实使用** 兼顾，避免百科全书式堆砌。
- **输出**：规则一句话 + 正例 / 反例 + 与中文思维差异提示（如适用）。

### 4.5 著名英语文献与名著（Literature exposure）

- **定位**：**导读与语言学习**，非文学专业全文研究。
- **内容类型**：
  - **公版作品**：可节选较短段落（长度上限由实现与版权策略决定），附词汇注释与难句分析。
  - **仍在版权保护期作品**：以 **情节摘要、主题讨论、名言引用（简短）、仿写练习** 为主，避免大段复制现代畅销书原文。
- **来源说明**：若使用联网检索，应在回答中区分「模型摘要」与「网页摘要」，并提醒用户以正式出版物为准。

### 4.6 按需实时学习资料（Dynamic materials）

- **触发**：用户给出主题（如「人工智能伦理」「气候变化」）、体裁（说明文 / 议论文 / 对话）、目标时长（阅读 3 分钟）、难度。
- **生成策略**：
  1. 优先检查用户 **知识库（RAG）** 是否已有相关笔记或教师提供的材料。
  2. 若无或不足，由 **大模型生成** 分级短文或对话。
  3. 若用户需要 **最新事实或热点用法**，调用 **互联网检索工具**，将结果整理为 **学习友好** 的短文或注释列表（非原始长文堆砌）。
- **质量约束**：标注不确定信息；避免编造新闻细节；引用网页时尽量简要。

### 4.7 实时翻译（Real-time translation）

- **场景**：用户粘贴英文或中文段落，要求翻译、润色、或双语对照。
- **呈现形式**：
  - **整段译文**（适合快速理解）；
  - **逐句或逐句块对照**（适合精读）；
  - **术语表**：专名、多义词在本文中的取义。
- **边界声明**：回答中应提示「重要文件请人工校对」；医学法律等敏感场景建议寻求专业服务。

## 5. 与现有 Agent 能力的对齐关系（不写具体代码）

以下基于 `services/agent` 常见形态归纳，实现时以仓库现状为准。

| 能力 | 英语学习中的用途 |
|------|-------------------|
| **流式 SSE** | 翻译与讲解的实时输出 |
| **会话记忆 / 摘要** | 多轮同一主题的连续学习与跟进 |
| **知识库 RAG** | 用户自建生词本、摘录、老师材料的个性化复现 |
| **互联网检索** | 补充名著背景、新闻英语、用法核实（摘要级） |
| **系统提示（专项模式）** | 可通过专项 `assistMode` 或等价机制附加「英语学习守则」（实现阶段定义） |

## 6. 合规、版权与安全

- **版权**：名著正文节选须控制长度与来源；现代作品以导读与原创练习为主。
- **未成年人**：若产品面向青少年，应在界面层提示合理使用时长与家长关怀（具体文案由产品与法务定）。
- **内容安全**：拒绝仇恨、违法与明显不适宜的学习「素材」请求；政治敏感话题以 **中性语言学习角度** 处理或礼貌拒绝，由实现策略细化。

---

## 7. 验收标准（MVP 与增强）

### 7.1 MVP（可对外宣称「英语学习助手」的最低标准）

- [ ] 用户可选择或说明难度后，获得 **单词 / 短语 / 句子 / 语法** 中至少三类结构化输出。
- [ ] 支持 **中英互译** 与 **至少一种对照格式**（整段或分句）。
- [ ] 能基于用户指令生成 **一篇分级短文或对话** 供阅读。
- [ ] 能结合 **知识库** 复习用户自有材料（若有入库）。
- [ ] 流式输出可用；回答包含 **水平分层** 与 **边界提示**（翻译免责声明等）。

### 7.2 增强（后续迭代，非 MVP 阻塞）

- 持久化「学习计划 / 错题本 / 复现间隔」。
- TTS（文本转语音）跟读、发音评分对接第三方。
- 按体裁（学术 / 商务）的模板化写作教练。
- 专项词汇书或真题库的授权引入（商务与数据合规另行 SPEC）。
