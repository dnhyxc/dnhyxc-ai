## 2. 账号与访问控制

- **路由级登录守卫**（更新：2026-04-05）：未登录时，访问受保护页面会被引导到登录页；公开页面不受影响。
- **鉴权失效收敛**（更新：2026-04-05）：接口返回 401（未授权）时统一清理会话并触发重新登录，避免“看似已登录但实际已过期”的状态漂移。
- **公开访问路径策略**（更新：2026-05-14）：未登录仍可访问首页、登录、关于、分享链接、**知识库**（默认仅本地模式，见 **§6**）、系统设置及其子路径、桌面端下载说明页、产品指南、用户服务政策、用户服务协议、结构化更新说明等；智能对话等需登录的主流程仍由守卫拦截；独立公开页与 **§21、§22** 所列路由一致。
- **头像与聊天附件迁至腾讯云 COS**（更新：2026-05-26）：账户头像与智能对话中的聊天附件均改由服务端上传至腾讯云对象存储，不再使用七牛前端直传或应用服务器本地目录。上传成功后保存完整对象地址；Web 端通过同源 **`/ext-cos/`** 代理展示，避免混合内容拦截；大图预览与附件下载已适配，并在成功或失败时给出提示。部署需在服务端配置 COS 密钥与桶信息，并保证对象可被正常读取（推荐公有读或等价策略）。
- **Stripe 会员充值（三档套餐）**（更新：2026-06-08）：登录后可在 **`/pay`** 选择**月度（¥9.9）/ 季度（¥25.9）/ 年度（¥99.9）** 固定套餐，价格在页面内**不可自行修改**；通过 Stripe 嵌入式收银台完成支付。支付成功后会开通或续期会员（在原有到期日上叠加时长），并自动进入 **`/profile` 个人主页** 查看会员徽章与有效期。会员到期后登录或访问资料时会自动校正为非会员；资料页对非会员展示开通引导文案。
- **会员开通幂等修复**（更新：2026-06-08）：修复同一笔 Stripe 支付因 Webhook 与支付完成回调并发而**重复叠加会员时长**的问题（例如月度套餐到期日多显示约一个月）。新支付不再重复加时；若历史账号已多算，需联系运维在后台校正到期日。
- **登录后立即被踢出修复**（更新：2026-06-08）：修复「云端朗读参数账号同步」上线后，部分账号**登录成功瞬间又被判未登录、跳回登录页**的问题（后台同步朗读偏好时凭证尚未就绪）。现已调整登录顺序与会员预拉取策略，线上可正常保持登录态。
- **切换账号时清空本地缓存**（更新：2026-06-08）：在同一浏览器内**登出、换号登录**或会话**401 失效**时，会自动清空与上一账号相关的前端展示缓存，包括知识库**未保存草稿**、文档**助手 / RAG 对话**、**英语学习 Agent** 聊天与进行中的单词包/经典句进度等，避免误看到他人内容。同一账号仅更新资料或会员状态（用户 id 不变）时不会清空。
- **个人主页会员标识优化**（更新：2026-06-08）：**`/profile` 个人主页**上，有效会员的「会员」徽章与「会员有效期至 …」文案改为**高对比金色**展示，深色背景下更易识别；会员状态判定逻辑在前端统一维护，与大模型设置页的会员默认策略保持一致。

- **分享会话顺序修复**（更新：2026-04-22）：修复分享页中消息顺序错乱的问题，提升阅读一致性。
- **联网检索与引用**（更新：2026-04-02）：支持联网检索并在输出中携带引用信息，便于溯源与复核。
- **附件与 OCR**（更新：2026-04-02）：支持附件处理与 OCR（光学字符识别）增强，提升多模态输入的可用性。
- **异步落库与可靠性**（更新：2026-04-02）：通过队列（BullMQ）等机制提高消息持久化的可靠性与可扩展性。
- **桌面端对话输入区：语音与停录策略**（更新：2026-05-02）：桌面客户端（Tauri）对话页底部输入区支持文本/语音模式切换、实时听写与停录收尾；停录后不再发起整段二次转写，详见 **§11**。
- **对话模型接入统一**（更新：2026-05-21）：主站智能对话的后端生成链路统一为硅基流动 OpenAI 兼容接口，默认使用 **GLM-5.1** 系列模型；流式输出、停止生成、续写与分支等交互方式不变。
- **聊天附件图片预览修复**（更新：2026-05-25）：修复 Web 与桌面端上传图片后预览失败（含中文文件名、跨端口拦截、线上网关配置不当等）。Web 生产环境附件通过当前站点下的接口地址加载，不再依赖单独配置图片静态路径；发消息时附件路径与服务器磁盘文件名一致，便于识别图片内容。请同时部署前端与后端并重启服务；若仍直接访问旧式图片链接，需按运维文档调整网关。
- **图片附件文字识别**（更新：2026-06-13）：对话中上传的图片附件在发送前由服务端使用智谱视觉模型提取画面与文字描述，再交给对话模型回答；与你在设置里选择的对话大模型无关。自托管部署需在服务端配置智谱 API 凭证，否则图片类附件可能无法被正确理解。

## 6. 知识库（编辑、列表与本地模式）

- **云端与本地双模式**（更新：2026-04-05）：支持云端知识条目管理，也支持本地文件夹作为知识库来源。
- **未登录仅本地**（更新：2026-04-11）：未登录时默认使用本地模式，不会触发云端接口；并隐藏不适用的入口（如回收站）。
- **本地文件夹管理**（更新：2026-04-05）：支持递归扫描 Markdown 文件、读取/保存/删除，并支持在外部编辑器中打开进行编辑。
- **删除分流：本地 / 在线 / 同时删除**（更新：2026-04-23）：知识库列表删除确认框在「云端条目 + 桌面端已定位到同名本地文件」的场景下提供 **「删除本地文件 / 删除在线文件 / 同时删除」** 三按钮；同时保留原“同时删除”的既有行为，避免影响历史使用习惯。
- **自动保存（防抖）**（更新：2026-04-08）：提供自动保存能力（Debounce，防抖），避免频繁写入；并与“覆盖保存”语义联动，减少误覆盖与打断编辑。
- **页面内快捷键（Chord，和弦快捷键）**（更新：2026-04-14）：支持在知识库页面内使用组合快捷键完成保存、清空、打开列表、切换操作栏等，提高编辑效率。
- **回收站打开与清空后的编辑器状态**（更新：2026-04-16）：从回收站打开条目时，持久化快照与正文一致，Diff 基线正确；执行「新建/清空草稿」时会刷新编辑器会话标识，与从列表打开再清空的行为对齐，避免仍停留在分屏对照等视图状态。
- **知识库文档助手：置底/置顶与未登录隐藏**（更新：2026-04-21）：已登录时在知识库编辑器底部可使用文档助手；对话区支持一键**滚动到底部**或**回到顶部**（与 Markdown 预览区角标逻辑一致），便于长对话与流式输出时上滑阅读后再跟随最新内容；**未登录时不展示**该助手入口，避免无效占位。
- **助手流式跨文档稳定性**（更新：2026-04-22）：修复流式输出时切换文档/路由后再切回丢失流式状态的问题；并优化“首次保存文档时若仍在流式输出”的边界处理，避免误终止输出或绑定不完整会话。
- **助手输入区菜单随界面语言切换**（更新：2026-05-02）：知识库文档助手内底部输入与 **§12** 界面语言一致，「输入模式」等文案随中英切换；与 **§11** 桌面端语音能力配套使用。
- **本地目录与编辑器同步**（更新：2026-05-02）：本地文件夹扫描、写盘与编辑器缓冲及列表状态对齐；未登录时保持仅本地、不请求云端的策略与 **§6** 前述条目一致。
- **选区发送至文档助手**（更新：2026-05-14）：在知识库 Markdown 编辑器中可将**选中文本**发送至底部文档助手，结合 **AI 模式** 或 **RAG 模式** 提问；对重复或重叠选区的发送做收敛，降低助手侧重复上下文噪音。
- **「生成目录」写入文首带二级标题**（更新：2026-05-21）：知识库 AI 模式下点击助手 **「生成目录」** 快捷卡，流式结束后自动插入的目录块**最上方**统一为 **`## 目录`**，其下为锚点列表；若模型只返回列表或用了其它级别标题，写入编辑器时会自动规范化。若文首**已有锚点目录但缺少** `## 目录`，再次生成时**只补标题**、不重复插入列表；文首已是 `## 目录` 则提示跳过。
- **知识库助手流式体验**（更新：2026-05-21）：AI 模式助手回复不再展示模型的 **「思考过程」** 折叠区（与主站对话一致）；流式时 **「正在生成中…」** 旁的加载图标可正常旋转。
- **知识库保存前自动格式化**（更新：2026-05-21）：手动保存与防抖自动保存在落盘前会先按编辑器内「格式化文档」规则整理 Markdown（含代码围栏安全处理），再写入云端或本地文件。
- **知识库向量化入库稳定性**（更新：2026-06-13）：修复部分文章保存后云端向量化失败（如接口 404 或参数无效）导致检索不可用的问题；长文与非会员默认向量模型会按档位自动缩短分片，减少入库报错。
- **知识库 RAG 多向量库检索**（更新：2026-06-13）：在设置中启用自定义向量配置后，知识库 RAG 会**并行检索**你曾保存过的全部向量库，并**始终包含**系统默认 bge 库，切换向量模型后旧库中的文章仍可被检索到；新文章仍写入当前选定的向量库。
- **知识库全站仅 BGE 与入库稳定性**（更新：2026-06-13）：超级管理员可在大模型设置中开启「仅使用 BGE 向量库」，开启后全员知识库向量化与 RAG 统一走系统 BGE 库与模型；修复全站 BGE 下长文（含 emoji）保存向量化失败，以及 BGE 小分片导致单次写入过大的问题。
- **会员默认向量库纳入 RAG**（更新：2026-06-13）：有效会员在已保存自定义向量后，RAG 除你的向量库与系统 bge 库外，会自动并行检索会员默认 Qwen3 向量库，减少漏检存量。
